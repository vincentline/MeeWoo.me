# 新页面 - Avatar图标预览器需求文档

## 项目概述
创建一个新的静态网页应用，用于预览和组合Avatar图标素材，支持多层图片叠加预览和导出。

## 页面结构

### 1. 预览区
- 尺寸：750px * 750px
- 实现方式：Canvas
- 默认底色：#F7F7F7
- 图片层级（从下到上）：
  - 底层：按钮B上传的图片
  - 中间层：身体图片（无、男、女）
  - 顶层：按钮A上传的图片
- 所有图片显示尺寸：750px * 750px

### 2. 图片操作功能
- 支持鼠标滚轮缩放图片（缩放步进5%）
- 支持鼠标左键拖动图片
- 默认图片居中显示
- 当图片不是默认状态时，预览区下方显示"恢复"按钮
- 点击"恢复"按钮可重置图片位置和缩放比例

### 3. 操作提示
- 位于预览区下方
- 提示用户如何操作编辑预览区图片

### 4. 身体图片选择
- 选项：无、男、女
- 实现方式：3个按钮，点击出现选中状态
- 默认选中：无
- 选中男/女时，预览区显示对应身体图片
- 选中无时，不显示身体图片

### 5. 底色选择
- 选项：默认、肤色
- 实现方式：2个按钮，点击出现选中状态
- 默认选中：默认（底色#F7F7F7）
- 选中肤色时，Canvas底色变为肤色（#FFDAB9）

### 6. 按钮A
- 功能：上传图片（不需要裁切功能）
- 上传后图片显示在预览区顶层
- 上传图片后，按钮右边出现"恢复"按钮
- 点击"恢复"按钮：
  - 恢复按钮消失
  - 移除预览区通过按钮A上传的图片

### 7. 按钮B
- 功能：上传图片（不需要裁切功能）
- 上传后图片显示在预览区底层
- 上传图片后，按钮右边出现"恢复"按钮
- 点击"恢复"按钮：
  - 恢复按钮消失
  - 移除预览区通过按钮B上传的图片

### 8. 生成图标按钮
- 功能：点击后下载素材图标
- 文件名规则：`btn_` + 按钮A的素材图名称

## 交互流程

1. **初始状态**：
   - 预览区显示#F7F7F7底色
   - 无任何图片
   - 身体图片选择默认"无"
   - 底色选择默认"默认"
   - 按钮A和按钮B无恢复按钮

2. **上传图片A**：
   - 点击按钮A选择图片
   - 图片显示在预览区顶层
   - 按钮A右侧出现"恢复"按钮

3. **上传图片B**：
   - 点击按钮B选择图片
   - 图片显示在预览区底层
   - 按钮B右侧出现"恢复"按钮

4. **选择身体图片**：
   - 点击"男"或"女"按钮
   - 对应身体图片显示在预览区中间层

5. **选择底色**：
   - 点击"肤色"按钮
   - Canvas底色变为#FFDAB9

6. **操作图片**：
   - 鼠标滚轮缩放图片（步进5%）
   - 鼠标左键拖动图片
   - 预览区下方显示"恢复"按钮

7. **恢复操作**：
   - 点击预览区下方的"恢复"按钮：图片回到默认位置和缩放比例
   - 点击按钮A/B右侧的"恢复"按钮：移除对应图片

8. **生成图标**：
   - 点击"生成图标"按钮
   - 下载组合后的图片，文件名为`btn_` + 按钮A的素材图名称

## 技术要求

### 前端技术栈
- HTML5 + CSS3 + JavaScript（原生开发）
- Canvas API用于图片显示和操作
- 沿用index页面的设计风格

### 界面设计
- 简洁现代的UI设计
- 响应式布局，适配不同屏幕尺寸
- 清晰的操作流程和视觉反馈
- 与index页面保持一致的设计风格

## 实现要点

1. **Canvas分层绘制**：
   - 底层：按钮B的图片
   - 中间层：身体图片
   - 顶层：按钮A的图片

2. **图片操作**：
   - 实现鼠标滚轮缩放（步进5%）
   - 实现鼠标拖动功能
   - 维护每个图片的位置和缩放状态

3. **状态管理**：
   - 跟踪每个图片的上传状态
   - 跟踪身体图片的选择状态
   - 跟踪底色选择状态
   - 跟踪图片操作状态（是否需要显示恢复按钮）

4. **文件下载**：
   - 组合Canvas中的所有图层
   - 生成PNG图片
   - 按指定命名规则下载

## 注意事项

- 所有图片都以750*750尺寸显示
- 默认图片居中
- 支持缩放和拖动操作
- 提供恢复功能
- 下载文件名格式为`btn_` + 按钮A的素材图名称
