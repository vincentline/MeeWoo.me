# Avatar图标生成器 - 需求文档

## 项目概述
创建一个静态网页应用，用于上传图片并按1:1比例拆切生成avatar图标。

## 功能需求

### 1. 图片上传功能
- 页面初始状态显示一个大按钮："点击上传图片生成avatar图标"
- 支持上传单个或多个PNG格式图片
- 只允许上传PNG格式文件

### 2. 图片拆切功能
- 上传图片后弹出拆切弹窗（尺寸约800*800）
- 使用第三方拆切工具（推荐使用Cropper.js）进行图片拆切
- 支持功能：
  - 拖动图片调整位置
  - 滚轮缩放图片（缩放步进2%）
  - 显示1:1比例的拆切区域框
  - 固定1:1拆切比例
  - 自动对齐拆切边缘（可选）
- 确定后生成1:1比例、198px*198px尺寸的PNG图片

### 3. 多图片处理
- 如果上传多个图片，按顺序依次弹出拆切弹窗
- 每个图片拆切完成后显示在页面上

### 4. Canvas显示与导出
- 使用Canvas展示拆切后的图片
- Canvas尺寸与图片尺寸一致（198px*198px）
- Canvas背景色固定为#F7F7F7
- 下载的图片必须包含#F7F7F7底色

### 5. 下载功能
- 每个Canvas下方有单独的下载按钮
- 支持单个图片下载
- 多图片时，列表底部有"打包下载"按钮
- 使用zip.js等压缩库将多张图片打包成ZIP文件下载

### 6. 状态管理
- 有Canvas显示时，列表左上角显示"返回"按钮
- 点击返回按钮回到初始空白状态

## 技术要求

### 前端技术栈
- HTML5 + CSS3 + JavaScript
- Canvas API用于图片显示和导出
- 第三方图片拆切库（需调研选择合适的库）
- zip.js用于多图片打包下载

### 界面设计
- 简洁现代的UI设计
- 响应式布局，适配不同屏幕尺寸
- 清晰的操作流程和视觉反馈

## 交互流程

1. **初始状态**：
   - 页面显示上传按钮
   - 无任何Canvas和下载按钮

2. **上传图片**：
   - 用户点击上传按钮选择PNG图片
   - 系统验证文件格式
   - 弹出拆切弹窗

3. **图片拆切**：
   - 用户在弹窗中调整图片位置和拆切区域
   - 使用第三方工具进行1:1比例拆切
   - 点击确定后生成198px*198px图片

4. **图片展示**：
   - 拆切后的图片显示在Canvas中
   - Canvas下方显示下载按钮
   - 列表左上角显示返回按钮

5. **多图片处理**：
   - 多个图片按顺序拆切
   - 所有拆切后的图片在页面上垂直排列

6. **下载操作**：
   - 单个下载：点击对应Canvas下方的下载按钮
   - 批量下载：点击列表底部的打包下载按钮

7. **返回操作**：
   - 点击返回按钮清空所有Canvas和下载按钮
   - 回到初始状态

## 实现要点

1. **第三方拆切工具实现**：
   - 使用Cropper.js库实现图片裁剪功能
   - 配置固定1:1裁剪比例
   - 实现滚轮缩放（缩放步进2%）
   - 支持拖动图片调整位置
   - 显示裁剪区域框
   - 可选：自动对齐裁剪边缘

2. **Canvas操作**：
   - 创建固定尺寸（198px*198px）的Canvas
   - 设置Canvas背景色为#F7F7F7
   - 将拆切后的图片绘制到Canvas上
   - 支持从Canvas导出图片

3. **文件下载**：
   - 单个图片下载：通过Canvas.toDataURL()实现
   - 多图片打包：使用zip.js库将多个图片文件打包成ZIP

4. **状态管理**：
   - 维护上传图片列表
   - 跟踪当前处理状态
   - 管理Canvas元素的创建和销毁

## 开发计划

1. 搭建基础项目结构
2. 实现图片上传功能
3. 集成第三方拆切工具
4. 实现Canvas显示和单个下载功能
5. 实现多图片处理和打包下载功能
6. 添加状态管理和返回功能
7. 优化UI和交互体验
8. 测试和修复问题

## 注意事项

- 确保Canvas背景色正确设置为#F7F7F7
- 下载的图片必须包含背景色
- 只支持PNG格式的图片上传和导出
- 拆切比例必须严格保持1:1
- 最终生成的图片尺寸必须为198px*198px
