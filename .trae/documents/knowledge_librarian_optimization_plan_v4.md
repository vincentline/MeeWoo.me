# Knowledge Librarian 体验优化方案 v4.0

> **背景**: 尽管 v3.0 建立了基本的自动化流水线，但实际高频使用中暴露出“批量处理能力弱”、“决策指引模糊”、“反馈机制缺失”等痛点。本方案旨在解决这些“最后一公里”的体验问题。

## 1. 核心优化：批量处理 (Batch Processing)

**问题**: `archiver.py` 只能单文件操作，导致面对大量碎片时 Agent 需进行数十次工具调用，效率低下且容易耗尽 Context。

**解决方案**: 升级 `archiver.py` 支持 `--batch` 模式。

### 1.1 新增 `batch` 命令

允许一次性传入多个文件，或者指定 `--all` 处理整个 Inbox。

* **调用方式**:

  * `python archiver.py batch --all` (全量自动归档)

  * `python archiver.py batch --files "file1.md,file2.md"` (指定列表)

* **智能分组逻辑**:

  * 脚本内部预读文件内容（提取关键词/Tags）。

  * 根据 **3.2 领域分类标准** 自动将文件分组 (e.g., Group 1 -> Graphics, Group 2 -> UI)。

  * **交互式确认**: 脚本输出分组建议，Agent 确认后一次性执行。

  * *注*: 如果自动分组太难实现，至少支持 `batch_create`，让 Agent 指定一个 Target，脚本把多个 Source 合并进去。

### 1.2 进度条反馈

* 在 CLI 输出中增加总体进度：`[Batch] Processing 3/24: konva-drag.md ... ✅`

## 2. 认知增强：决策指引 (Decision Guide)

**问题**: Agent 在选择模板、验证命令、归档策略时存在疑惑，缺乏明确标准。

**解决方案**: 在 `SKILL.md` 中增加 **决策速查表 (Cheat Sheet)**。

### 2.1 模板选择指南

| 模板类型        | 适用场景                          | 示例                      |
| :---------- | :---------------------------- | :---------------------- |
| `new`       | **通用/默认**。不确定时用这个。            | 任何模块                    |
| `guide`     | **操作教程**。包含步骤 (Step 1, 2, 3)。 | "如何配置 Vite", "环境搭建"     |
| `concept`   | **原理/概念**。解释 "是什么/为什么"。       | "双缓冲机制", "Vue 响应式原理"    |
| `reference` | **API/速查**。包含参数表、接口定义。        | "Konva API 列表", "错误码字典" |

### 2.2 归档策略指南 (Create vs Merge)

* **Create**:

  * 目标模块**不存在**。

  * 目标模块**已存在但体积过大** (>300行)，需要拆分新文件。

  * 内容**自成一体**，与其他规则关联度低。

* **Merge**:

  * 目标模块**已存在且体积适中**。

  * 内容是现有规则的**补充/修正/特例**。

### 2.3 验证命令示例 (Verify Cmd)

* **配置检查**: `grep "outDir" vite.config.js`

* **依赖检查**: `grep "konva" package.json`

* **文件存在**: `ls src/components/MyComponent.vue`

## 3. 脚本修复：闭环与反馈

**问题**: 索引更新失效、健康检查无行动。

**解决方案**:

### 3.1 修复 `clean_inbox.py`

* **强制同步**: 确保删除文件后，立即重写 `index.md`。

* **容错**: 如果 `index.md` 格式不标准（如缺少表格线），脚本应尝试修复或追加到末尾，而不是静默失败。

### 3.2 增强 `check_health.py`

* **新增** **`--fix`** **参数**:

  * 对于超大文件，自动创建子目录 (e.g., `modules/graphics/canvas/`) 并移动原文件为 `index.ts.md`，为拆分做准备。

  * 对于格式错误的 Interface，尝试自动包裹。

***
