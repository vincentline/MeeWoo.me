# Trae 工具高级使用技巧指南

> 本文档总结了 Trae IDE 提供给 Agent 的工具的高级使用技巧，旨在帮助 Agent 更高效、更智能地完成任务。

---

## 一、代码搜索与探索工具高级技巧

### SearchCodebase 高级技巧
- **语义化搜索优化**：使用具体的功能描述而非简单关键词，如 "用户登录认证逻辑" 而非 "login"
- **跨文件关联搜索**：结合多个相关概念进行搜索，如 "React 组件状态管理 + 表单验证"
- **上下文理解**：在搜索时提供项目背景信息，如 "在 Vue 3 项目中实现路由守卫"
- **结果筛选**：利用返回的代码片段上下文，快速定位最相关的实现

### Grep 高级技巧
- **正则表达式优化**：使用更精确的正则模式，如 `function\s+\w+\s*\(.*\)\s*\{` 匹配函数定义
- **多文件类型搜索**：结合 `glob` 参数搜索多种文件类型，如 `**/*.{ts,tsx,js,jsx}`
- **上下文控制**：使用 `-A`、`-B`、`-C` 参数获取足够的代码上下文
- **负向匹配**：使用 `!` 排除不需要的文件，如 `!**/node_modules/**`

### Glob 高级技巧
- **递归搜索**：使用 `**` 进行深度递归搜索，如 `src/**/*.ts`
- **多模式组合**：使用 `{}` 匹配多种模式，如 `*.{js,ts,json}`
- **排除模式**：结合 `.gitignore` 逻辑，避免搜索无关文件

## 二、文件操作工具高级技巧

### Read 高级技巧
- **选择性读取**：对于大文件，使用 `offset` 和 `limit` 参数精确定位代码片段
- **分块读取**：对于超大文件，分多次读取不同区域
- **文件类型识别**：根据文件扩展名自动调整读取策略
- **编码处理**：注意处理不同编码的文件，确保正确读取

### Write 高级技巧
- **模板化写入**：使用预设模板快速生成标准文件结构
- **增量更新**：结合 Read 操作，只修改必要部分
- **文件类型适配**：根据文件类型调整内容格式（如 JSON、XML、代码文件）
- **错误处理**：捕获写入异常，确保文件操作安全

### Edit 高级技巧
- **精确匹配**：使用足够的上下文确保只修改目标代码
- **批量替换**：使用 `replace_all` 进行全局变量重命名或模式替换
- **代码重构**：结合多个 Edit 操作实现复杂代码重构
- **版本控制**：在编辑前读取文件内容，确保基于最新版本修改

---

## 三、命令执行工具高级技巧

### RunCommand 高级技巧
- **环境配置**：在执行命令前设置必要的环境变量
- **工作目录管理**：根据任务类型自动切换到合适的工作目录
- **命令组合**：使用 PowerShell 语法组合多个命令，如 `cd path; command1; command2`
- **非阻塞模式**：启动服务器时设置 `blocking: false`，并通过 `CheckCommandStatus` 监控状态
- **输出处理**：处理命令输出，提取关键信息

### CheckCommandStatus 高级技巧
- **输出过滤**：使用 `filter` 参数过滤出需要的信息
- **状态监控**：定期检查长时间运行命令的状态
- **错误检测**：监控命令执行过程中的错误信息
- **输出分页**：处理大量输出时使用 `skip_character_count` 进行分页

## 四、任务管理工具高级技巧

### TodoWrite 高级技巧
- **任务分解**：将复杂任务分解为具体、可执行的子任务
- **优先级设置**：根据任务重要性和依赖关系设置合理优先级
- **状态管理**：及时更新任务状态，确保任务执行的连续性
- **进度跟踪**：通过任务状态变化跟踪整体进度
- **任务关联**：在任务描述中添加相关文件路径或功能模块信息

### Skill 高级技巧
- **技能组合**：根据任务类型组合使用多个技能，如 `coder` + `webapp-testing`
- **技能参数优化**：根据具体场景调整技能参数
- **技能链**：构建技能执行链，如先使用 `knowledge-gardener` 提取经验，再使用 `coder` 实现功能
- **自定义技能**：根据项目需求创建自定义技能

## 五、交互与诊断工具高级技巧

### AskUserQuestion 高级技巧
- **问题设计**：设计清晰、具体的问题，避免模糊表述
- **选项提供**：为用户提供合理的选项，包含推荐选项
- **上下文关联**：在问题中包含相关背景信息
- **多问题组合**：在一个请求中组合多个相关问题
- **默认值设置**：为常见问题设置合理的默认答案

### GetDiagnostics 高级技巧
- **文件定位**：针对特定文件获取诊断信息
- **错误分类**：根据错误类型优先级处理问题
- **修复建议**：基于诊断结果提供具体的修复建议
- **持续监控**：在修改代码后重新获取诊断，确保问题解决

## 九、综合使用技巧

### 工具组合策略
- **搜索 -> 分析 -> 执行**：先使用搜索工具了解代码结构，再分析问题，最后执行解决方案

- **任务管理 -> 技能调用**：使用 TodoWrite 规划任务，再调用相应技能执行
- **诊断 -> 修复 -> 验证**：获取诊断信息，修复问题，再验证修复效果

### 效率提升技巧
- **并行操作**：同时执行多个不相关的工具调用
- **缓存利用**：缓存搜索结果和常用命令
- **模板使用**：创建常用操作的模板
- **自动化流程**：构建自动化工作流程，减少重复操作
- **错误处理**：建立完善的错误处理机制

### 智能决策技巧
- **上下文理解**：充分理解项目上下文和用户需求
- **优先级判断**：根据任务重要性和紧急性安排工作
- **风险评估**：评估操作风险，制定应对策略
- **资源优化**：合理分配和使用系统资源
- **持续学习**：从每次操作中学习，不断改进工作方式

## 十、最佳实践案例

4. 使用 `RunCommand` 运行测试

4. 使用 `RunCommand` 验证修复效果

### 案例三：功能开发

5. 使用 `GetDiagnostics` 检查代码
6. 使用 `webapp-testing` 测试功能
