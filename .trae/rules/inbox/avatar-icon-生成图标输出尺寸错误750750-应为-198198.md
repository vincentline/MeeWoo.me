# avatar-icon 生成图标输出尺寸错误（750×750 应为 198×198）
> Tags: avatar-icon,konva,canvas,icon-resize
> Created: 2026-05-22

## Context
avatar-icon 页面的"生成图标"按钮预期输出 198×198 PNG（从 750×750 画布内容缩小），但实际输出的是 750×750 原尺寸图片。

## Root Cause
`generateIcon()` 方法在 [preview.js:L568](src/gadgets/avatar-icon/js/preview.js) 中调用 `exportStage.toDataURL({ pixelRatio: 1 })` 获取 750×750 的 data URL 后，直接触发下载，缺少从 750×750 → 198×198 的缩小步骤。

## Solution
在获取 750×750 data URL 后，通过 `new Image() + canvas 2D` 将其缩小到 198×198，再导出下载：

```javascript
const iconSize = 198;
const img = new Image();
img.onload = () => {
  const canvas = document.createElement('canvas');
  canvas.width = iconSize;
  canvas.height = iconSize;
  const ctx = canvas.getContext('2d');
  ctx.drawImage(img, 0, 0, iconSize, iconSize);
  // ... 下载 canvas.toDataURL('image/png')
};
img.src = dataURL;
```

关键点：先渲染 750×750 保留画布布局，再等比缩小到 198×198，避免直接改 Stage 尺寸影响渲染结果。
